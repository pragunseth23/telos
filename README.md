# Telos MVP (Phase 1)

Telos is a goal-aware assistant centered on an Intent Graph that maps identity, long-horizon goals, actionable tasks, constraints, and tensions.

This implementation targets the Phase 1 scope in `project_spec.md`:

- Conversation-only onboarding driven by OpenAI model
- Intent Graph construction (root, Speed-2, Speed-1)
- Directed graph edges with weighted nodes and conflict metadata
- Temporal decay metadata and graph version history snapshots
- Manual Speed-1 task labeling (`Agent`, `Human`, `Hybrid`)
- Single-agent execution for Speed-1 tasks
- Intent Alignment Report for every execution
- Persistent memory (graph, chat, logs) in browser local storage
- Persistent model context file on disk (Tauri app data directory)
- Minimalist UI with qualitative intent signals (no exposed numeric weight/confidence/reward values)
- All chat replies (onboarding + workspace) are generated by OpenAI via native Rust commands

## Tauri Desktop Run

1. Install prerequisites:
- Rust toolchain
- Tauri CLI (`cargo install tauri-cli`)
- Platform prerequisites from [Tauri v2 docs](https://v2.tauri.app/start/prerequisites/)

2. Create `.env` from `.env.example` and set `OPENAI_API_KEY`.

3. Start desktop app:

```bash
npm run tauri:dev
```

This launches the native Tauri window and all chat/model calls run via Rust commands.

## Secrets & Context

- Keep keys only in `.env` (never in frontend code).
- `.env` is ignored by git via `.gitignore`.
- Persistent model memory is stored in Tauri app data:
  - macOS: `~/Library/Application Support/com.telos.desktop/context/context.json`
  - Windows: `%APPDATA%/com.telos.desktop/context/context.json`
  - Linux: `~/.local/share/com.telos.desktop/context/context.json`
- Use the UI Reset button to clear browser state and persisted model context.

## Architecture

- `index.html`: app entrypoint
- `src-tauri/src/main.rs`: native OpenAI integration + context file persistence commands
- `src-tauri/tauri.conf.json`: Tauri app configuration
- `styles.css`: responsive 3-panel UI and visual system
- `src/app.js`: onboarding + workspace UI + orchestration
- `src/api.js`: frontend API client for model turns
- `src/intentGraph.js`: graph engine and objective/reward logic
- `src/agent.js`: single-agent execution and alignment report generation
- `src/memory.js`: browser persistence
- `tests/*.test.js`: core graph and agent behavior tests

## Product Notes

- Irreversible actions require explicit confirmation before execution.
- `Human` tasks are blocked from agent execution by policy.
- API keys stay in Rust side and are never exposed to browser JS.
- Model conversation context is persisted by Tauri in app data directory:
  - macOS: `~/Library/Application Support/com.telos.desktop/context/context.json`
  - Windows: `%APPDATA%/com.telos.desktop/context/context.json`
  - Linux: `~/.local/share/com.telos.desktop/context/context.json`
- Version history is captured for transparency after graph mutations.
